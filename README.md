Данный проект является финальной работой по модулю «Автоматизация тестирования». В рамках проекта реализованы UI- и API-автотесты для проверки ключевой функциональности сервиса Кинопоиск.

## 📑 Документация
- **Финальная работа по ручному тестированию: [https://superteamsky.yonote.ru/share/f190a281-5352-4511-940d-11a7cd42696c]

## 🛠 Технологии и инструменты
- **Язык:** Python 3.10+
- **Тестовый фреймворк:** Pytest
- **UI-тесты:** Selenium WebDriver
- **API-тесты:** Requests
- **Отчетность:** Allure Report
- **Линтеры:** Flake8 (PEP8)
- **Архитектура:** Page Object Model (POM)

## 📁 Структура проекта
- `pages/` — Page Object классы для UI-тестов.
- `tests/` — папка с тестами:
    - `test_ui.py` — UI-автотесты (5+ кейсов).
    - `test_api.py` — API-автотесты (5+ кейсов).
- `data/` — конфигурационные файлы и тестовые данные.
- `conftest.py` — фикстуры для настройки браузера и предусловий.
- `requirements.txt` — список зависимостей.


## Настройка окружения
Для работы API-тестов необходимо создать файл `.env` в корневой папке проекта и добавить в него ваш API-ключ от Кинопоиска:

1. Создайте файл `.env`
2. Добавьте в него строку:
   API_KEY=ваш_персональный_ключ
   SESSION_ID=ваш_session_id_из_браузера
   SESSION_ID2=ваш_sessionid2_из_браузера

1. **Установка зависимостей:**
   ```bash
   pip install -r requirements.txt
2. **Запуск (выберите нужный режим):**
   Все тесты: pytest
   При прохождении тестов использовать ручное прохождение капчи
   Только UI: pytest -m ui
   Только API: pytest -m api
3. **Генерация отчета:**  
   pytest --alluredir=allure-results
   allure serve allure-results
